## Description
Implement the BrottBrain behavior card evaluation engine as defined in architecture.md section 3.5.

## Requirements
- `brottbrain.gd` — main evaluation engine
  - `stance`: current active stance
  - `cards`: Array[BehaviorCard] (max 8, priority ordered)
  - `evaluate(brott, enemies, arena)` → Optional action
- `behavior_card.gd` — card data structure
  - `trigger`: TriggerCard (condition)
  - `action`: ActionCard (response)
- Each tick, iterate cards top→bottom. First matching trigger fires its action.
- If no card fires, follow current stance defaults.

### Trigger Types (minimum for Sprint 2)
- `hp_below(threshold)` — my HP below %
- `hp_above(threshold)` — my HP above %
- `enemy_hp_below(threshold)` — enemy HP below %
- `enemy_in_range(range)` — enemy within distance
- `energy_above(threshold)` — my energy above %
- `energy_below(threshold)` — my energy below %
- `always` — always true (catch-all)

### Action Types (minimum for Sprint 2)
- `set_stance(stance)` — switch to stance
- `move_to_cover` — find nearest cover, move there
- `hold_position` — stop moving
- `focus_fire(target_selection)` — target lowest HP enemy
- `conserve_energy` — only fire cheapest weapon

## Acceptance Criteria
- [ ] BrottBrain evaluates cards in priority order per tick
- [ ] Triggers evaluate correctly against game state
- [ ] Actions produce correct movement/combat overrides
- [ ] Falls back to stance defaults when no card matches
- [ ] Max 8 cards enforced
- [ ] Unit tests for trigger evaluation and card priority
- [ ] Clean GDScript following architecture patterns
