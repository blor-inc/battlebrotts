## Description
Implement the arena tile system and line-of-sight raycasting as defined in architecture.md sections 3.3 and 3.6.

## Requirements
- `arena_manager.gd` — manages the tile grid (32×32 px tiles)
- Tile types: floor, wall (blocks LoS + movement), cover (50% miss chance, destructible 50HP), pillar (indestructible, blocks LoS + movement), hazard (lava: 10 dmg/sec ignoring armor)
- LoS raycasting from Brott center to target center on tile grid
- Cover detection: if ray passes through cover tile, 50% miss chance per shot
- Arena data structure that pathfinding and combat systems can query
- At least one arena layout ("The Pit" — 16×16 open arena)
- Environment effects: lava damage applied in tick system

## Acceptance Criteria
- [ ] Arena tile grid with configurable dimensions
- [ ] Tile types: floor, wall, cover, pillar, hazard
- [ ] LoS raycast returns {blocked: bool, cover_count: int}
- [ ] Cover blocks are destructible (take damage, removed at 0 HP)
- [ ] Lava tiles deal 10 dmg/sec (0.5/tick) ignoring armor
- [ ] "The Pit" arena layout defined (16×16, open with some cover)
- [ ] Unit tests for LoS through clear, wall, and cover tiles
- [ ] Clean GDScript following architecture patterns

## Reference
- docs/architecture.md sections 3.3, 3.6
- Tile size: 32×32 px
- Arena sizes: 16×16 to 20×20 tiles
