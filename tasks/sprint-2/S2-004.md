## Description
Implement the four stance movement behaviors as defined in architecture.md section 3.3.

## Requirements
- `steering.gd` — stance-based movement behaviors
- Each stance produces a target position that the pathfinder navigates to

### Stances
1. **Aggressive**: Move toward nearest enemy, close to weapon range
2. **Defensive**: Retreat to nearest cover tile, maintain max weapon range
3. **Kiting**: Maintain 60-80% of max weapon range, circle-strafe clockwise
4. **Ambush**: Move to cover, hold position, fire when enemy enters 50% weapon range

### Integration
- BrottBrain actions can temporarily override stance movement
- Stance determines default behavior when no BrottBrain card fires
- Movement executes at chassis speed via pathfinder

## Acceptance Criteria
- [ ] All 4 stances produce correct target positions
- [ ] Aggressive closes distance to weapon range
- [ ] Defensive seeks cover and maintains max range
- [ ] Kiting maintains 60-80% range with circle-strafe
- [ ] Ambush finds cover and holds until enemy in range
- [ ] BrottBrain overrides work correctly
- [ ] Unit tests for each stance behavior
- [ ] Clean GDScript following architecture patterns
